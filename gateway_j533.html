Gateway J533 tap — Seat Altea XL 2011 (PQ35)

Ghid montaj ESP32 + SN65HVD230 pentru listen-only pe CAN-Antrieb prin conectorul gateway-ului J533.
Identificat din VCDS: SW 7N0 907 530 C • HW 1K0 907 951 • Gateway Highline H37 • Platforma PQ35 (1P family).

Obiectiv: montare discreta, listen-only, zero risc, cu acces la toate semnalele broadcast de pe CAN-Antrieb: RPM, torque, load (ECU) • wheel speeds individuale FL/FR/RL/RR (ABS) • yaw rate, lateral G, brake pressure (ESP) • steering angle precis • status ABS/ESP/TCS interventii.

1. Arhitectura CAN pe PQ35 — confirmare

Gateway J533 face bridge intre 4 magistrale CAN + cluster CAN pe unele variante. CAN-Infotainment (de unde citeste acum SYU CAN box-ul din HU) primeste mesaje retransmise partial de gateway — de asta wheel speeds, yaw rate, brake pressure nu apar acolo. Trebuie listenat direct pe sursa: CAN-Antrieb.

Diagrama: toate CAN-urile converg la gateway; noi ne legam in paralel pe CAN-Antrieb fara a rupe firul.

2. Localizare fizica Gateway J533 pe Altea XL 2011

Unde este

Zona: sub bord, stanga, in spatele panoului lateral dash (side trim kick panel) de la piciorul soferului, adiacent sau sub blocul sigurantelor interior.

Reper Seat Altea 2006-2015 (confirmat in documentatia portal-diagnostov):

"Below the dash panel on left, near centre console" — pozitie 805/28 in wiring manual oficial.

Aspect: cutie plastica neagra cu autocolant cu barcode + part number + label H37.

Conector: singur, multi-pin, tipic rosu sau negru (20-pin T20 pe 1P/5P Altea).

Cum ajungi la el

  1. Deschide usa soferului, scoate pragul plastic de pe soseaua stangii (clip-uri simple, trage in sus).
  2. Scoate panoul end-dash de pe lateralul bordului (unde iese cablajul din tavan). Clip plastic + eventual surub Torx T20.
  3. Priveste in sus si spate, spre coloana volanului. Modulul e fixat pe brat metalic sau pe un suport dedicat.
  4. Alternative: stai pe spate in zona pedalelor, lumineaza in sus cu lanterna / foto cu telefonul — e metoda documentata de comunitatea comma.ai pentru instalarea harness-ului.

Tips hardware importante

  • Dezlipeste R120 de pe modulul SN65HVD230 (unele board-uri chinezesti au terminator 120Ω intre H si L, care dezechilibreaza bus-ul existent). Verifica cu multimetrul: daca intre pinii CAN_H si CAN_L citesti ~120Ω cu modulul nealimentat → are terminator, scoate-l.
  • Firele la T-tap scurte si rasucite (twisted pair, max 20 cm), altfel apar reflectii pe 500kbps.
  • LM2596 cu protectie: condensator 100µF la intrare + dioda TVS SMBJ18A pe +12V (standard automotive pentru spike-uri la demaror).
  • Alimentare ACC, nu B+: ESP32 cu WiFi activ consuma ~150mA permanent → baterie descarcata in 2 zile. ACC se stinge cu cheia.
  • Consum standby J533: <1mA (bus sleep dupa 30s). Atentie sa nu-l trezesti permanent cu tapul tau daca transceiver-ul iese din silent mode.

5b. Alimentare ESP32 direct din gateway (T30 + siguranta sticla)

Solutia practica si curata: alimentezi ESP32 tot de la conectorul T20 gateway, fara sa tragi fire suplimentare. Pin 1 si 2 sunt ambii T30 (+12V permanent), iar pin 14 e T31 (masa). Se ia de pe pin 1, prin siguranta fir din sticla 5x20mm, 1A (inline fuse holder), apoi in LM2596.

Bilant consum

Total practic cand masina merge: ~80 mA din linia T30 — siguranta 1A are rezerva 12x. Perfect ok.

Linia T30 pe J533: alimentata din blocul sigurantelor interior prin siguranta F5 de 5A (dupa wiring Altea). Consumul nostru = 1.6% din capacitate — zero impact pe gateway.

Siguranta recomandata

  • Tip: siguranta sticla 5x20mm, rapid-blow, 1A (nu 2A — ESP32 nu are voie sa traga asa mult, daca trage e scurt).
  • Holder: inline waterproof cu fire flexibile (gen "ATP auto fuse holder 5x20mm" de pe AliExpress ~1 EUR, sau Emag "portsiguranta fir 5x20").
  • Pozitionare: cat mai aproape de punctul de taping pe fir, nu aproape de ESP32 — protejeaza cablul, nu doar placa.
  • Alternative acceptabile: mini-blade auto 1A (ATC / MINI) cu holder. Sau rezistor PTC autoresetabil 1.1A (MF-R110) in loc de siguranta — nu arde daca da scurt ocazional.

Strategia drain-baterie

Problema: T30 e permanent. Daca ESP32 ramane in WiFi AP cu masina oprita → 150mA continuu → bateria 60Ah descarcata la 40% in ~1 saptamana (plus consumul propriu al masinii). Inacceptabil.
Solutia: firmware-ul ESP32 implementeaza deep-sleep cu wake-on-CAN:
  • La startup ESP32 porneste TWAI + WiFi AP.
  • Daca 60s nu vede nici un frame CAN (= masina oprita, bus asleep) → esp_deep_sleep_start().
  • GPIO wake-up de pe CRX (RX al transceiver-ului) → orice activitate pe bus trezeste chip-ul in ~50ms.
  • Consum in sleep: ~10µA → bateria tine ani de zile fara drain perceptibil.
  • La pornirea masinii → bus se trezeste → ESP32 se trezeste → AP+WiFi up in ~2 secunde (timp reboot).

Schema alimentare actualizata

Schema completa — totul se ia din conectorul T20 al gateway-ului: alimentare (pin 1), masa (pin 14), semnal (pin 6 + 16).

6. Procedura pas-cu-pas

  1. Pregatire: masina oprita, cheie scoasa, usa inchisa 60 secunde ca bus-ul sa intre in sleep. Deschizi usa soferului si actionezi imediat — ai ~30s pana se trezeste bus-ul Komfort.
  2. Demontare panou: scoti pragul stanga si kick panel lateral. Localizezi J533.
  3. Scoti conectorul T20: apasa clemia de siguranta, trage drept. NU trage de fire.
  4. Verificare pinout cu multimetru (conector deconectat, masa referinta = caroserie):
    • Rezistenta intre pin 6 si pin 16: ~60Ω (2x 120Ω in paralel = terminatori ECU + gateway) → confirmat Antrieb.
    • Daca citesti ~120Ω → e un singur terminator, posibil bus partial / diferit (repeta masurare cu alt pereche pini ca 5-15 pt Komfort).
    • Daca citesti 0Ω sau ∞ → pini gresiti, opreste-te si re-verifica schema.
  5. Verificare osciloscop (cu conector reconectat, contact pornit sau motor pornit):
    • Pe pin 16 (CAN-H) fata de masa: linie baseline ~2.5V, pulsuri scurte pana la ~3.5V.
    • Pe pin 6 (CAN-L) fata de masa: linie baseline ~2.5V, pulsuri scurte scaderi la ~1.5V.
    • Trigger pe diferential H-L: burst-uri dense de bit la 2µs (=500kbps). Asta confirma definitiv Antrieb.
    • Pentru comparatie: Komfort (pin 15-5) are bit-time 10µs (=100kbps) — mai lent vizibil.
  6. T-tap hardware: folosesti conectori Scotchlok sau sonde IDC (insulation displacement). Eu recomand sonda tip Test-Klemme (clema tip crocodil fin cu ac care patrunde prin izolatie fara taiere). Dupa test, inlocuiesti cu Scotchlok permanent sau cablu sudat + tub termo-retractabil.
  7. Fixare ESP32: cutie plastica mica (3D print sau dozatoare electronice ~5x7cm), fixata cu velcro industrial pe brat metalic langa J533.
  8. Test aprins: contact pornit (nu motor), cu telefonul la AP ErgissVAGCAN si navigand la http://192.168.4.1/data — trebuie sa vezi frame-uri curgand (~500/sec).
  9. Logging 60s cu motor pornit, variind tura + viraje usoare, sa surprinzi toate ID-urile de interes. Dump binary sau JSON salvat pt analiza.
  10. Mapping ID-uri: corelare cu VCDS measuring blocks. Fiecare ID CAN de interes se valideaza observand ca valoarea se schimba sincron cu masurarea VCDS.

8. Firmware ESP32 — schelet

Folosesc library-ul ESP32-TWAI-CAN (Arduino). Scheletul urmator este pentru Faza A (dump raw pe AP WiFi):

#include <ESP32-TWAI-CAN.hpp>
#include <WiFi.h>
#include <WebServer.h>

#define CAN_TX       5
#define CAN_RX       4
#define BUF_SIZE     256
#define SLEEP_AFTER  60000UL   // ms — intra in deep sleep daca bus tace 60s

CanFrame buf[BUF_SIZE];
volatile uint16_t head = 0, tail = 0;
uint32_t lastFrameMs = 0;

WebServer srv(80);

void enterDeepSleep() {
  Serial.println("Bus idle - deep sleep, wake on CRX activity");
  // GPIO 4 (CRX) cu pullup, wake la nivel low (CAN dominant bit)
  esp_sleep_enable_ext0_wakeup((gpio_num_t)CAN_RX, 0);
  WiFi.mode(WIFI_OFF);
  esp_deep_sleep_start();
}

void setup() {
  Serial.begin(115200);
  ESP32Can.setPins(CAN_TX, CAN_RX);
  ESP32Can.setRxQueueSize(128);
  ESP32Can.setSpeed(TWAI_SPEED_500KBPS);
  ESP32Can.setMode(TWAI_MODE_LISTEN_ONLY);   // CRITIC - nu emite nimic
  if (!ESP32Can.begin()) { Serial.println("CAN init FAIL"); while(1); }

  WiFi.softAP("ErgissVAGCAN", "vagcan2026");
  lastFrameMs = millis();
  srv.on("/data", []() {
    String j = "[";
    uint16_t i = tail; bool first = true;
    while (i != head) {
      if (!first) j += ",";
      j += "{\"id\":" + String(buf[i].identifier) +
           ",\"dlc\":" + String(buf[i].data_length_code) + ",\"d\":[";
      for (int k = 0; k < buf[i].data_length_code; k++) {
        if (k) j += ",";
        j += String(buf[i].data[k]);
      }
      j += "]}";
      i = (i+1) % BUF_SIZE; first = false;
    }
    tail = head;
    j += "]";
    srv.send(200, "application/json", j);
  });
  srv.begin();
}

void loop() {
  CanFrame f;
  while (ESP32Can.readFrame(f, 0)) {
    buf[head] = f;
    head = (head+1) % BUF_SIZE;
    if (head == tail) tail = (tail+1) % BUF_SIZE; // overflow discard
    lastFrameMs = millis();
  }
  srv.handleClient();

  if (millis() - lastFrameMs > SLEEP_AFTER) {
    enterDeepSleep();   // wake-on-CAN via GPIO4
  }
}

Faza B (dupa mapping) va avea filtering hardware + parsare pe ID-urile de interes + output JSON structurat.

9. Integrare in ErgissVAG Android app

In D:/apps/ergissvag/app/src/main/java/ro/ergiss/vag/ adaug WifiCanReader.kt paralel cu SyuCanReader.kt:

  • Detectie automata AP ErgissVAGCAN → conectare silentioasa.
  • Poll HTTP http://192.168.4.1/data la 200ms (5Hz — suficient pt HUD).
  • Prioritate sursa in ObdService:
    • Wheel speeds, yaw rate, lateral G, brake pressure, steering precis → WifiCanReader (Antrieb, sursa originala)

10. Checklist siguranta

  • Masina oprita si 60s in sleep la fiecare demontare/remontare conector.
  • Multimetrul verifica ~60Ω H↔L inainte de orice conectare.
  • Osciloscop confirma bit-time 2µs = 500kbps Antrieb.
  • Transceiver in mod LISTEN_ONLY obligatoriu (TWAI_MODE_LISTEN_ONLY).
  • Terminator 120Ω de pe modul SN65HVD230 dezlipit.
  • Twisted pair intre T-tap si transceiver <20cm.
  • Alimentare din pin 1 gateway (T30) cu siguranta sticla 1A inline (nu direct — risc scurt = arde firul).
  • TVS diode 18V pe linia 12V inainte de LM2596.
  • Firmware implementeaza deep-sleep wake-on-CAN — ESP32 intra in sleep la 60s fara trafic (masina oprita) → consum 10µA → zero drain baterie.
  • Test initial pe bench (nu in masina) cu OBD sau cu generator CAN sintetic.
  • Dupa montaj — test VCDS scan. Nu trebuie sa apara DTC-uri noi. Eventuale "U" code-uri (comunicatie) = probleme hardware de rezolvat inainte de utilizare permanenta.
